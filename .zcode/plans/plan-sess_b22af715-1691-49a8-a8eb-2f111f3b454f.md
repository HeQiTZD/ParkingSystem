# 配置文件架构重构计划

## 目标
将 `InitFile` 改为真正的单例，消除多实例内存副本冲突、析构无意义写盘、DB 层绕过封装、ConfigInitDialog 信号断链四大问题。

---

## 修改清单（按文件）

### 1. `src/utils/initfile.h` — 单例化

- 构造函数从 `public` 改为 `private`
- 新增 `public: static InitFile& instance();`
- 删除拷贝/移动构造与赋值（`= delete`）
- 析构函数保持 public（静态局部变量自动析构），但移除函数体内的 `saveConfig()` 调用
- 更新头文件尾部注释中"通常采用单例模式"的描述，改为"已采用单例模式"

### 2. `src/utils/initfile.cpp` — 单例实现 + 移除析构保存

- 新增 `instance()` 实现：
  ```cpp
  InitFile& InitFile::instance() {
      static InitFile instance;
      return instance;
  }
  ```
- 修改 `~InitFile()`：移除 `saveConfig()` 调用，改为空析构体（或仅打印一条析构日志）。注释说明"保存由 setter 调用方显式触发，析构不自动写盘"
- 其余方法（loadConfig / saveConfig / getter / setter / validateConfig / getDefaultConfig）保持不变

### 3. `src/app/main.cpp` — 改用单例

- 删除 `InitFile initFile;` 栈变量声明（main.cpp:55）
- `loadConfigWithRetry(&initFile)` → `loadConfigWithRetry(&InitFile::instance())`（main.cpp:68）
- `initFile.getDbHost()` 等 5 个 getter → `InitFile::instance().getDb*()`（main.cpp:72-74）
- `initFile.loadConfig()` → `InitFile::instance().loadConfig()`（main.cpp:88）
- `QObject::connect(&initFile, ...)` → `QObject::connect(&InitFile::instance(), ...)`（main.cpp:58）
- `MySQLInit mysqlInit(&dbManager, &initFile)` → `MySQLInit mysqlInit(&dbManager, &InitFile::instance())`（main.cpp:95）
- 信号 lambda 中 `Q_UNUSED(freeMinutes)` 保持不变，修正注释明确"freeMinutes 仅存配置文件，不同步 DB"

### 4. `UI/MainWindow/mainwindow.cpp` — 消除局部 InitFile 实例

四处全部改为 `InitFile::instance()` 直接读内存（单例已在 main 中加载，无需重复 loadConfig）：

- **onExitSearchButton**（main.cpp:282-286）：
  ```cpp
  // 旧：InitFile cfg; cfg.loadConfig(); ...
  // 新：
  double cost = Car::calculateFee(intTime, outTime,
      InitFile::instance().getParkingPrice(),
      InitFile::instance().getFreeMinutes());
  QString parkingName = InitFile::instance().getParkingName();
  ```

- **onUpdateParkingCount**（mainwindow.cpp:312-321）：
  ```cpp
  // 旧：InitFile initFile; if(!initFile.loadConfig()){...} ...
  // 新：
  QString name = InitFile::instance().getParkingName();
  if(name.isEmpty()){
      qWarning() << "配置文件中停车场名称为空";
      return;
  }
  ```

- **getParkingNameFromConfig**（mainwindow.cpp:337-345）：
  ```cpp
  // 旧：InitFile cfg; if(!cfg.loadConfig()){...} return cfg.getParkingName();
  // 新：return InitFile::instance().getParkingName();
  ```

### 5. `UI/ConfigInit/configinitdialog.cpp` — 复用单例，打通信号

- `saveConfig()`（configinitdialog.cpp:137-155）中 `InitFile initFile;` → `InitFile::instance()`
  ```cpp
  // 新：
  InitFile::instance().setDbConfig(...);
  InitFile::instance().setParkingConfig(...);  // 发射 parkingConfigChanged
  InitFile::instance().saveConfig();
  ```
- 移除 `initFile.loadConfig()`（单例已在 main 中加载）
- **效果**：`setParkingConfig` 发射的 `parkingConfigChanged` 信号现在能到达 main.cpp:58 连接的 lambda → `dbManager.updateParkingConfig()`，配置窗口的改动自动同步到数据库

### 6. `src/database/databasemanager.cpp` — 删除直接读 JSON，改用单例

- `deleteCarRecord`（databasemanager.cpp:505-511）中直接读 config.json 的代码：
  ```cpp
  // 旧：
  QString path = QCoreApplication::applicationDirPath() + "/config.json";
  QFile file(path);
  if(!file.open(QIODevice::ReadOnly)) return false;
  QJsonDocument doc = QJsonDocument::fromJson(file.readAll());
  file.close();
  QString parkingName = doc.object()["parking"].toObject()["name"].toString();

  // 新：
  QString parkingName = InitFile::instance().getParkingName();
  ```
- 文件头部添加 `#include "src/utils/initfile.h"`
- 移除不再需要的 `<QJsonDocument>` / `<QJsonObject>` / `<QCoreApplication>` / `<QFile>` include（如果仅此处使用；若其他方法也用到则保留）

### 7. `src/database/mysqlinit.cpp` / `mysqlinit.h` — 保持不变

- `m_initFile` 指针继续通过构造函数注入，只是 main.cpp 传入的指针现在指向单例实例
- 接口不变，零改动

## 不在本次范围内（保持现状）

1. **camera / recognition / system 配置节未接入** — CameraThread 硬编码 index=0，这些配置节目前无消费者，属于功能缺失而非架构缺陷，留待后续设置页面接入
2. **ApplicationManager 窗口 parent=nullptr 与注释矛盾** — 属另一独立问题，不在本次重构范围
3. **freeMinutes 双真相源** — 维持现状（仅存配置文件），修正注释说明设计意图
4. **`qt.core.qmetaobject.connectslotsbyname: No matching signal for on_miniButton_clicked()`** — UI 控件命名问题，与配置架构无关

---

## 预期效果

重构后日志中反复出现的 `配置文件路径 → 配置文件加载成功 → 配置文件保存成功` 三连将大幅减少：
- 启动时：仅 1 次 loadConfig（main.cpp），不再有每次业务操作的重复加载
- 运行时：入库/出库/刷新车位只读内存，**零磁盘 IO**
- 配置保存仅在 ConfigInitDialog 提交时发生（合理时机）
- ConfigInitDialog 修改停车场参数后，自动同步到数据库（信号打通）

## 验证方式

1. 编译通过（`qmake && make` 或 Qt Creator 构建）
2. 运行程序，观察日志：
   - 启动时仅出现 1 次"配置文件加载成功"
   - 入库/出库操作后**不再**出现"配置文件保存成功"
   - 退出时**不再**出现"配置文件保存成功"
3. 功能验证：
   - 首次运行（无 config.json）→ 弹出配置窗口 → 填写并提交 → 数据库 PARKING 表数据正确
   - 已有 config.json → 正常启动 → 入库/出库 → 费用计算正确
   - 删除车辆记录 → parkingName 读取正确（验证 databasemanager.cpp 改动）

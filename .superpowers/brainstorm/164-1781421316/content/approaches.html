无边框窗口设计方案

ConfigInitDialog - 选择最适合的实现方式

A

纯 QSS + 事件过滤器（推荐）

直接在 UI 文件中添加按钮，用 QSS 样式化，通过事件过滤器实现拖拽。阴影使用 QGraphicsDropShadowEffect。

优点

  • 实现简单，代码量少
  • 与现有 config.qss 风格统一
  • 无需额外类或文件

缺点

  • 拖拽逻辑与 Dialog 耦合
  • 不可复用到其他窗口
B

自定义标题栏组件

创建独立的 TitleBar QWidget，包含拖拽区域和按钮。可复用到 MainWindow 等其他窗口。

优点

  • 模块化，可复用
  • 职责分离清晰
  • 便于统一管理标题栏样式

缺点

  • 需要新建类和文件
  • 初期工作量稍大
C

原生 Windows 阴影方案

使用 DwmExtendFrameIntoClientArea 实现系统级窗口阴影，外观最接近原生窗口。

优点

  • 阴影效果最自然
  • 性能最优

缺点

  • 仅限 Windows 平台
  • 需要调用 Windows API
  • 与 Qt 样式系统不兼容

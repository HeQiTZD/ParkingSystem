实现方案对比

选择 QSS 文件的组织方式

A

方案A：独立配置文件（推荐）

创建单独的 styles/config.qss，在 ConfigInitDialog 构造函数中加载

优点

  • 样式独立，易于维护
  • 可复用于其他配置窗口
  • 不影响全局样式

缺点

  • 需要额外的文件加载代码
B

方案B：合并到全局样式

将 ConfigInitDialog 样式添加到 styles/global.qss 末尾

优点

  • 只需维护一个文件
  • 自动随全局样式加载

缺点

  • global.qss 会变得臃肿
  • 不同窗口样式耦合
C

方案C：内联到 UI 文件

在 configinitdialog.ui 的 widget 属性中直接设置样式

优点

  • 所见即所得
  • 无需额外文件

缺点

  • 难以复用和维护
  • 样式与布局耦合
  • 不支持伪类选择器

# 摄像头设置界面需求文档

**版本:** 1.0
**日期:** 2026-07-21
**状态:** 已实现

---

## 1. 概述

为停车场管理系统的「摄像头管理」页面提供摄像头参数配置弹窗。用户可通过该弹窗配置全局采集参数和每路摄像头的独立属性。

### 1.1 适用范围

| 包含 | 不包含 |
|------|--------|
| 全局分辨率 / 帧率配置 | 数据库配置 |
| 每路摄像头名称、位置、角色、分辨率、帧率 | 停车场参数 |
| 角色互斥（entry 唯一性约束） | 识别置信度配置 |
| 配置持久化至 JSON 文件 | 摄像头增删操作 |

### 1.2 触发入口

摄像头管理页面 (`CameraManagement`) 的「设置」按钮。

---

## 3. 功能需求

### 3.1 全局采集参数

| 字段 | 控件类型 | 预设值 | 说明 |
|------|----------|--------|------|
| 分辨率宽度 | QComboBox（可编辑） | 480 / 640 / 800 / 1024 / 1280 / 1920 / 2560 | 默认值 1920 |
| 分辨率高度 | QComboBox（可编辑） | 同上 | 默认值 1080 |
| 帧率 | QComboBox（可编辑） | 15 / 20 / 25 / 30 | 默认值 30，单位 FPS |

- 打开弹窗时从 `InitFile` 加载当前配置并回显。
- 保存时写入 `InitFile`，重启应用后生效。

### 3.2 摄像头列表

- **数据来源：** `CameraManager::instance()` 获取摄像头数量和每路信息。
- **无摄像头时：** 显示 "未检测到摄像头" 居中提示文字。
- **动态生成：** 每路摄像头生成一行编辑区域，包含以下字段：

| 字段 | 控件类型 | 说明 |
|------|----------|------|
| 标题 | QLabel | "📷 摄像头 N"（N 从 1 开始） |
| 名称 | QLineEdit | 默认值来自 CameraInfo.name，为空时回退为 "摄像头 N" |
| 位置 | QLineEdit | 默认值来自 CameraInfo.location |
| 角色 | QRadioButton × 2 | "entry"（入口）/ "monitor"（监控），默认来自 CameraInfo.role |
| 分辨率宽度 | QComboBox（可编辑） | 预设同全局，默认值来自 CameraInfo.width |
| 分辨率高度 | QComboBox（可编辑） | 预设同全局，默认值来自 CameraInfo.height |
| 帧率 | QComboBox（可编辑） | 预设 15/20/25/30，默认值来自 CameraInfo.fps |

### 3.3 角色互斥约束

- 所有摄像头的 role 单选按钮属于同一互斥组。
- **保存时校验：**
  - 多于 1 个 "entry" → 弹窗阻止保存，提示 "只能设置一个入口摄像头（entry）"。
  - 0 个 "entry" → 警告确认弹窗 "当前无入口摄像头，车牌识别功能将暂停。确定继续？"，用户确认后可保存。
  - 恰好 1 个 "entry" → 正常保存。

### 3.4 输入校验

- 全局/每路分辨率宽度、高度、帧率必须为正整数。
- 输入非数字或 ≤0 → 弹窗提示具体错误，阻止保存。
- 名称为空 → 保存时自动回退为 "摄像头 N"。

### 3.5 保存操作

1. 校验全局参数（正整数）。
2. 校验角色互斥。
3. 遍历所有摄像头行校验每路参数（正整数）。
4. 读取控件值构建 `QJsonArray`。
5. 调用 `InitFile::setCameraConfig()` 写全局配置。
6. 调用 `InitFile::setCameras()` 写每路配置。
7. 调用 `InitFile::saveConfig()` 持久化。
8. 保存成功 → 发射 `camerasUpdated()` 和 `cameraConfigChanged()` 信号 → 提示 "摄像头配置已保存，重启应用后生效" → 关闭弹窗。
9. 保存失败 → 弹窗提示 "配置保存失败"。

### 3.6 取消操作

点击「取消」或「×」按钮 → 直接关闭弹窗，不保存任何修改。

## 4. 窗口行为

| 特性 | 实现 |
|------|------|
| 窗口类型 | `Qt::FramelessWindowHint \| Qt::WindowMinimizeButtonHint` |
| 背景 | `Qt::WA_TranslucentBackground`（半透明背景） |
| 投影 | `QGraphicsDropShadowEffect`，模糊半径 20，颜色 rgba(0,0,0,80) |
| 拖拽 | 标题栏 `eventFilter` 捕获 MousePress/Move/Release 实现拖拽 |
| 样式 | 加载 `:/styles/camerasettings.qss`，作用域 `#CameraSettingsDialog` |
| 模态 | 默认模态对话框 |

## 5. 数据模型

### 5.1 CameraInfo 结构

```cpp
struct CameraInfo {
    int     index    = 0;       // 摄像头序号
    QString name;               // 名称
    QString location;           // 位置
    QString role;               // "entry" | "monitor"
    int     width    = 1920;    // 分辨率宽度
    int     height   = 1080;    // 分辨率高度
    int     fps      = 30;      // 帧率
};
```

### 5.2 JSON 持久化格式

```json
{
  "cameraWidth": 1920,
  "cameraHeight": 1080,
  "cameraFps": 30,
  "cameras": [
    {
      "index": 0,
      "name": "主入口",
      "location": "大门",
      "role": "entry",
      "width": 1920,
      "height": 1080,
      "fps": 30
    }
  ]
}
```

## 6. 信号接口

| 信号 | 触发时机 | 接收方 |
|------|----------|--------|
| `camerasUpdated()` | 配置保存成功后 | CameraManagement 刷新摄像头名称/位置/角色显示 |
| `cameraConfigChanged()` | 配置保存成功后 | CameraManagement 提示用户重启使配置生效 |

---

## 7. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 无可用摄像头 | 全局区正常显示，摄像头列表显示 "未检测到摄像头" |
| 分辨率/帧率输入非法值 | 保存时校验，弹窗提示具体字段错误 |
| 角色互斥冲突 | 保存时校验，弹窗阻止并提示 |
| 0 个入口摄像头 | 警告确认后允许保存 |
| 配置文件写入失败 | 弹窗提示 "配置保存失败" |
| 名称为空 | 保存时自动填充默认名 "摄像头 N" |

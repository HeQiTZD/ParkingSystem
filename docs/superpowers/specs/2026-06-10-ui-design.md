# 停车场管理系统 UI 设计规范

**项目名称**：ParkingSystem（停车场管理系统）  
**设计日期**：2026-06-10  
**设计版本**：v1.0  
**设计风格**：现代简约风格

---

## 1. 设计概述

### 1.1 设计目标

- 采用现代简约风格，提升用户体验
- 以蓝色为主色调，体现专业感
- 左侧可折叠导航栏，优化空间利用
- 主页采用摄像头中心布局，突出核心功能

### 1.2 设计原则

- **简洁性**：界面简洁，无多余装饰元素
- **一致性**：统一的颜色、字体、间距规范
- **可用性**：操作流程清晰，反馈及时
- **可扩展性**：模块化设计，便于后续扩展

---

## 2. 整体架构

### 2.1 界面层次结构

```
QApplication
    ├── MainWindow (登录窗口)
    │   └── 居中登录表单
    │
    ├── RegisterWindow (注册窗口)
    │   └── 居中注册表单
    │
    ├── InitFileDialog (配置初始化窗口)
    │   └── 数据库配置表单
    │
    └── CarWindow (主功能窗口)
        ├── 左侧可折叠导航栏
        │   ├── 折叠状态 (60px)
        │   └── 展开状态 (200px)
        │
        └── 右侧内容区域 (QStackedWidget)
            ├── Page 0: 主页 (摄像头中心布局)
            ├── Page 1: 车辆信息
            ├── Page 2: 管理页面
            └── Page 3: 摄像头页面
```

### 2.2 页面切换逻辑

```
登录成功
    ↓
检查配置文件是否存在
    ↓
不存在 → 打开 InitFileDialog
    ↓
配置完成 → 进入 CarWindow (主功能界面)
    ↓
左侧导航栏切换页面
    ↓
退出 → 返回登录界面
```

### 2.3 核心组件关系

```
MainWindow (登录)
    ├── 注册按钮 → RegisterWindow
    ├── 登录按钮 → 验证成功 → CarWindow
    └── 数据库状态监听

CarWindow (主功能)
    ├── 导航栏按钮 → QStackedWidget 切换
    ├── 摄像头页面 → CameraThread
    ├── 主页 → PlateRecognize (车牌识别)
    └── 车辆信息 → DatabaseManager
```

- 过渡动画：宽度变化 200ms ease-in-out

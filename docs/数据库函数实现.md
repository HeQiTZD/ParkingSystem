## 1. Phase 2 — 车辆信息页面

### 1.0 `searchCars()` ✅ 已完成(2026-07-06)

**实现位置**: `src/database/databasemanager.cpp:341-421`

**核对结论**: 完全符合设计文档 Phase 2 要求。

| 设计要求 | 实现 |
|---------|------|
| 车牌模糊匹配(`LIKE`) | `"license_plate LIKE :plate"` + `"%"+plate+"%"` |
| 开始时间筛选 | `check_in_time >= :startTime`(仅 `isValid()` 时生效) |
| 结束时间筛选 | `check_out_time <= :endTime`(实现比文档更合理 — 对已出库车辆也能按出场时间筛) |
| 状态筛选 0/1/2 | `status==1 → check_out_time IS NULL` / `status==2 → IS NOT NULL` |
| 参数化查询防注入 | `prepare()` + 命名占位符 + `bindValue()` |
| 停车时长列 | `TIMESTAMPDIFF(MINUTE, check_in_time, IFNULL(check_out_time, NOW())) AS parking_minutes` |
| 状态列 | `CASE WHEN check_out_time IS NULL THEN '在场' ELSE '已离场' END AS status_text` |
| 返回列顺序 | `id, license_plate, check_in_time, check_out_time, fee, parking_minutes, status_text` |
| 排序 | `ORDER BY check_out_time DESC` |

**轻微偏差(非 bug)**:
- 结束时间筛选字段写的是 `check_out_time`(不是文档的 `check_in_time`)—— 更合理,保留
- 排序用 `check_out_time DESC`(不是 `check_in_time DESC`)—— 业务偏好,均可

### 1.1 `bool deleteCarRecord(int id)` ✅ 已完成(2026-07-06)

**实现位置**: `src/database/databasemanager.cpp:426-489`

**核对结论**: 4 处致命/建议修复已全部完成,函数可正常工作。

| 设计要求 | 实现 |
|---------|------|
| 删 CAR 记录 | ✅ `DELETE FROM CAR WHERE id = :id` |
| 在场时 -1 PARKING | ✅ `checkOutTime.isNull()` 判断(事务内实时查询,不信前端传参) |
| 防负数 | ✅ `GREATEST(P_now_count - 1, 0)` |
| 事务包裹 | ✅ `transaction()` + `commit()` / `rollback()` |
| 防 SQL 注入 | ✅ `prepare()` + `bindValue()` |

**轻微保留项(非 bug)**:
- 仍从 `config.json` 读 `parkingName` 作为 UPDATE 的 WHERE 条件 —— 单停车场系统可跑,多停车场时需重构为不带 WHERE 或 CAR 表冗余 `P_name` 字段

**作用**: 删除指定 ID 的车辆停车记录。

**参数**:

| 参数 | 类型 | 方向 | 说明 |
|------|------|------|------|
| `id` | `int` | 入参 | CAR 表主键 |
| 返回值 | `bool` | — | 成功/失败 |

**行为**:
- 若该车辆仍在场(`check_out_time IS NULL`),删除时同步更新 `PARKING.P_now_count - 1`
- 使用事务保证一致性
- 先用 `SELECT` 查在场状态,再决定是否需要更新车位统计

**设计文档依据**:
> Phase 2 设计:"选中行 → 确认对话框 → DELETE → 如果是在场车辆更新 PARKING 表"
>
> 原文:`DatabaseManager — searchCars(plate, start, end, status)、deleteCarRecord(id)`

**SQL 伪代码**:
```sql
-- 查是否在场
SELECT check_out_time FROM CAR WHERE id = ?;
-- 删除
DELETE FROM CAR WHERE id = ?;
-- 若在场才更新车位数
UPDATE PARKING SET P_now_count = GREATEST(P_now_count - 1, 0) WHERE P_name = ?;
```

### 4.1 前置:日志表设计(需在 `MySQLInit` 中补 DDL)

```sql
CREATE TABLE IF NOT EXISTS OPERATION_LOG(
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT COMMENT '操作用户ID',
    username VARCHAR(50) COMMENT '操作用户名(冗余/审计)',
    action VARCHAR(50) COMMENT '操作类型(check_in/check_out/login等)',
    detail TEXT COMMENT '操作详情(JSON 或文本)',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id(user_id),
    INDEX idx_action(action),
    INDEX idx_created_at(created_at)
```

# 停车场管理系统 — 代码架构设计文档
---

## 1. 项目概述

### 1.1 系统简介

基于 Qt/C++ 开发的桌面级停车场管理系统，集成 **HyperLPR** 深度学习车牌识别引擎，实现摄像头画面实时车牌检测、MySQL 数据持久化、多摄像头管理、用户权限分级、车辆进出记录查询、收费规则配置等完整业务闭环。

### 1.2 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Qt | 6.10.2 | UI 框架、信号槽、数据库访问、多媒体、图表、SVG |
| C++ | 11 | 核心语言 |
| MinGW | 13.1.0 | 编译器（项目强制指定） |
| MySQL | 8.x | 数据存储 |
| OpenCV | 4.11 | 图像处理、摄像头捕获（预编译库，项目携带） |
| HyperLPR | - | 车牌识别引擎（Caffe，源码编译，项目携带） |
| Qt Charts | - | 数据可视化 |
| Qt Multimedia | - | 摄像头多媒体采集 |
| QSS | - | UI 样式，按模块拆分独立文件 |

### 1.3 构建系统

- **构建工具:** qmake（主工程 `ParkingSystem.pro`）+ CMake（第三方依赖 `thirdparty/`）
- **编译器:** MinGW 13.1.0（硬编码 `C:/Qt/Tools/mingw1310_64/bin/g++.exe`）
- **构建方式:** 仅支持影子构建（Shadow Build），禁止源内构建
- **输出:** `ParkingSystem.exe`
- **编译优化:** OpenMP 并行加速（`-fopenmp`）、DLL 自动拷贝（`QMAKE_POST_LINK`）、编码强制 UTF-8

## 3. 分层架构

系统采用四层架构，层间通过 Qt 信号槽解耦：

- **表示层 (UI Layer):** MainWindow (QStackedWidget 导航容器) + 各功能页面
- **业务服务层 (Service Layer):** ParkingService / UserService / VehicleService
- **应用核心层 (App Core):** ApplicationManager / PlateRecognize / CameraManager / RecognizeThread / Car / FrameQueue / PlateConfirmTracker / ThreadPoolManager
- **数据访问层 (Data Layer):** DatabaseManager / DbConnectionPool / MySQLInit
- **MySQL 数据库:** User / CAR / PARKING 三张表

## 4. 核心模块详解

### 4.1 程序入口 (main.cpp)

**职责:** 应用程序初始化、配置加载、数据库连接、数据库初始化、服务创建、进入应用。

**启动流程:**

```
main()
  |
  +-> QApplication 创建
  |
  +-> InitFile::loadConfig() 加载 JSON 配置
  |   +-- 无配置 -> ConfigInitDialog 首次配置向导
  |   +-- 解析失败 -> 提示重试或退出
  |
  +-> DatabaseManager::connectDatabase() 连接 MySQL
  |   +-- 失败 -> 提示重配置或退出
  |
  +-> MySQLInit::initAll() 初始化表结构
  |   +-- User / CAR / PARKING 三表创建
  |   +-- 默认管理员账户 + 停车场基础数据
  |
  +-> 创建 ParkingService / UserService / VehicleService
  |
  +-> ApplicationManager::start()
  |   +-- LoginDialog 显示
  |   +-- 登录成功 -> 隐藏登录框 -> MainWindow 显示 + 启动识别
  |
  +-> app.exec() 事件循环
```

**关键文件:** `main.cpp` | `ApplicationManager.*` | `initfile.*` | `configinitdialog.*` | `mysqlinit.*`

### 4.2 ApplicationManager (应用状态管理器)

**职责:** 管理应用的全局生命周期：初始化 -> 登录 -> 主界面 -> 登出 -> 退出的状态流转。

**类结构:**

```cpp
class ApplicationManager : public QObject {
    Q_OBJECT
public:
    explicit ApplicationManager(DatabaseManager &dbMgr,
                                ParkingService &parkingSvc,
                                UserService &userSvc,
                                VehicleService &vehicleSvc,
                                QObject *parent = nullptr);
    void start();
private slots:
    void onLoginAccepted();      // 登录成功 -> 显示主窗口
    void onLogoutRequested();    // 登出 -> 返回登录界面
    void onAppExitRequested();   // 退出应用
private:
    DatabaseManager &m_dbMgr;
    ParkingService  &m_parkingSvc;
    UserService     &m_userSvc;
    VehicleService  &m_vehicleSvc;
    LoginDialog *m_loginDialog = nullptr;
    MainWindow  *m_mainWindow  = nullptr;
};
```

**生命周期管理:**
- **析构时** 依次 shutdown: RecognizeThread → ThreadPoolManager → CameraManager → PlateRecognize → InitFile
- **登录成功** `onLoginAccepted()`: 隐藏登录框 → 显示主窗口 → `startRecognition()` 启动识别
- **登出** `onLogoutRequested()`: 停止识别 → 隐藏主窗口 → 重置登录框 → 重新显示登录界面

### 4.5 PlateRecognize + RecognizeThread (车牌识别系统)

**PlateRecognize (单例):**

封装 HyperLPR 的 `pr::PipelinePR`，提供车牌识别接口。

```cpp
class PlateRecognize : public QObject {
    Q_OBJECT
public:
    static PlateRecognize& instance();
    bool loadModels(const QString &modelPath);
    int recognizePlate(const cv::Mat &image);
    int plateRecognizeAccessor(const cv::Mat &image, std::vector<PlateResult> &results);
    void shutdown();
signals:
    void recognizeFinished(cv::Mat plateImg, QString plateStr);
    void recognizeError(QString errorMsg);
    void modelLoadStatus(bool loaded, QString message);
};
```

**PlateResult 结构体:**

```cpp
struct PlateResult {
    cv::Mat plateImg;   // 车牌图像
    QString plateStr;   // 识别结果字符串
    float   score;      // 置信度
};
```

**RecognizeThread (独立识别线程):**

- 继承 QThread
- 从 `FrameQueue` 消费帧（`take()` 可设超时）
- 通过 `PlateRecognize::plateRecognizeAccessor()` 批量获取结果
- 通过 `PlateConfirmTracker` 过滤短期重复识别
- 发射 `plateRecognized(plate, plateImg)` 信号给 MainWindow

**FrameQueue (线程安全帧队列):**

```cpp
class FrameQueue {
public:
    void push(const cv::Mat &frame);
    bool take(cv::Mat &out, int timeoutMs);  // 阻塞取出
    void clear();
};
```

使用 `QMutex` + `QWaitCondition` 实现生产者-消费者模式。

### 4.6 PlateConfirmTracker (识别结果确认器)

**职责:** 对同一车牌连续多次识别才确认有效，并带冷却期防重复触发。

```cpp
class PlateConfirmTracker {
public:
    bool report(const QString &plate);
    void markActioned(const QString &plate);
    void setCooldownSecs(int secs);     // 触发后冷却期 (默认30s)
    void setConfirmCount(int count);    // 确认所需次数 (默认3次)
    void setExpirySecs(int secs);       // 过期清除 (默认5s)
};
```

**确认逻辑:** 同一车牌在 `expirySecs` 内被连续识别 `confirmCount` 次 → 确认有效。

### 4.8 DatabaseManager + DbConnectionPool + MySQLInit (数据层)

**DatabaseManager:**

封装所有 SQL 操作，按业务域分类：

**DbConnectionPool (单例):**

```cpp
class DbConnectionPool {
public:

    void setConfig(const DbConfig &config);

    void closeAll();
```

**MySQLInit:**

数据库表结构初始化，按依赖顺序创建：

1. `User` 表 — 用户账户

4. 默认管理员账户 + 停车场基础数据

## 7. 线程模型

### 7.1 线程分布

| 线程 | 职责 | 数量 |
|------|------|------|
| 主线程 (UI) | QApplication、MainWindow、所有 QWidget | 1 |
| CameraThread | 摄像头帧采集 (`cv::VideoCapture`) | 每路 1 个 |
| RecognizeThread | 车牌识别 (从 FrameQueue 消费) | 1 |

### 7.2 线程通信

- **CameraThread → MainWindow:** `newFrameCaptured` 信号 (QueuedConnection)
- **CameraThread → FrameQueue:** `push()` (线程安全)
- **RecognizeThread:** `take()` 从 FrameQueue 消费
- **RecognizeThread → MainWindow:** `plateRecognized` 信号 (QueuedConnection)

### 7.3 线程安全

- `std::atomic<bool>` 控制 CameraThread 运行/暂停状态
- `QMutex` + `QWaitCondition` 实现暂停等待
- `QMutex` 保护 `m_latestFrame` 跨线程访问
- DB 连接池按线程创建独立连接
